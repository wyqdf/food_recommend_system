## 当前实现快照

- 本地推荐开发口径：
  - 前端 `3000`
  - 后端 `8081`
  - 数据库优先导入完整 `food_recommend` 备份
- Windows 已提供源码直启与停止脚本：
  - `run-latest-dev.ps1`
  - `stop-latest-dev.ps1`
- 搜索现状：
  - 仓库默认配置值为 `search.engine=elasticsearch`
  - 本地推荐启动脚本会显式覆盖为 `SEARCH_ENGINE=auto`
  - ES 不可用或索引未就绪时，会回退到 MySQL 搜索链路
- 推荐现状：
  - 登录用户在命中场景且存在离线 `Top100` 时，`type=personal` 采用“分开筛选、再交错返回”
  - 目标配额按 `ceil(limit * 0.7)` 优先保证离线 `Top100`
  - 实时补位来自更大的热门源与非热门源候选池，并带随机性
  - 首页 `为你推荐` 与推荐页 `猜你喜欢` 使用同一条推荐链路
  - 首页 `热门菜谱` 改成从更大的热门候选池随机抽样
- 前端稳定性现状：
  - `Home.vue`
  - `Recommend.vue`
  - `Recipes.vue`
  - 上述页面都已使用 `latestRequest` 防止旧请求覆盖
- 最近一轮回归状态：
  - 最新源码脚本启动验证通过：前端 `3000`、后端 `8081`
  - 后端 `./mvnw.cmd test` 已通过，当前 `50` 项测试绿色通过
  - 前端 `npm run build` 已通过
  - 首页、搜索、详情、收藏、烹饪模式、投稿页、管理端首页与菜谱管理页已完成浏览器回归
  - 评论列表当前支持“匿名可读 + 登录态识别 `isLiked`”
