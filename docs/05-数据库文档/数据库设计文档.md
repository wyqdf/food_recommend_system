## 数据库概述

### 基本信息

- **数据库名称**: food_recommend
- **数据库版本**: MySQL 8.0
- **字符集**: utf8mb4 (支持 emoji 和特殊字符)
- **排序规则**: utf8mb4_unicode_ci
- **存储引擎**: InnoDB (支持事务和外键)

### 2026-03-18 当前实现说明

- 当前线上/本地运行中的管理员日志表为 `operation_logs`，不是旧文档中的 `system_logs`。
- 新增业务支撑表：
  - `user_preference_profiles`：冷启动问卷与用户偏好画像
  - `behavior_events`：行为埋点事件
  - `cooking_sessions`：烹饪会话进度、时长、完成状态
- 当前启动迁移会自动补充以下索引：
  - `idx_behavior_user_time`

### 2026-03-19 SQL 审计与索引优化现状

- 本轮只处理 `hot/warm` 查询，不对历史重复数据做清洗。
- 搜索主链路已切到 Elasticsearch，MySQL 搜索 fallback 仅保底，不作为本轮优化重点。
- 已新增的核心复合索引：
  - `comments`
    - `idx_comment_recipe_publish_time (recipe_id, publish_time DESC, id DESC)`
    - `idx_comment_create_time (create_time DESC)`
    - `idx_comment_user_create (user_id, create_time)`
  - `recipes`
    - `idx_recipes_status_create_time (status, create_time DESC, id DESC)`
    - `idx_recipes_status_like_count (status, like_count DESC, id DESC)`
    - `idx_recipes_status_rating_count (status, rating_count DESC, id DESC)`
  - `users`
    - `idx_users_status_create_time (status, create_time DESC, id DESC)`
  - `behavior_events`
    - `idx_behavior_user_time (user_id, create_time DESC, id DESC, recipe_id)`
    - `idx_behavior_user_event_time_recipe (user_id, event_type, create_time DESC, id DESC, recipe_id)`
    - `idx_behavior_user_time_scene (user_id, create_time DESC, scene_code)`
  - `interactions`
    - `idx_interactions_create_time_type (create_time DESC, interaction_type)`
- 已切换为 `INVISIBLE` 的历史冗余索引：
  - `comments.idx_comment_recipe`
  - `recipes.idx_createtime`
  - `recipes.idx_like_count`
  - `users.idx_users_status`
  - `users.idx_user_status`
  - `recipe_categories.idx_recipe_categories_recipe`
  - `recipe_ingredients.idx_ri`
- 当前验证结论：
  - 评论列表、用户端菜谱列表、管理端用户列表、最近行为菜谱取样均已命中新索引
  - 用户报告类聚合查询仍会使用临时表排序，但驱动索引已固定到合理路径
  - `idx_behavior_event_time`
  - `idx_behavior_recipe_time`
  - `idx_behavior_session_time`
  - `idx_cook_session_user_status_time`
  - `idx_cook_session_recipe_time`
  - `idx_cook_session_user_start_time`
  - `idx_recipe_author_uid`
- 已确认 `recipe_ingredients` 表存在历史重复数据：
  - 重复组合数：`2,200,804`
  - 额外重复行：`2,207,450`
  - 当前统计层已用 `COUNT(DISTINCT recipe_id)` 规避结果失真，但数据层尚未正式清洗

## 表结构详细说明

### 1. 管理员表 (admins)

**表名**: `admins`  
**说明**: 系统管理员信息表  
**引擎**: InnoDB  
**字符集**: utf8mb4

```sql
CREATE TABLE admins (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '管理员 ID',
    username VARCHAR(100) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(255) NOT NULL COMMENT '密码（BCrypt 加密）',
    email VARCHAR(100) COMMENT '邮箱',
    role ENUM('super_admin', 'admin') DEFAULT 'admin' COMMENT '角色',
    status TINYINT DEFAULT 1 COMMENT '状态：1-启用 0-禁用',
    last_login_time DATETIME COMMENT '最后登录时间',
    last_login_ip VARCHAR(50) COMMENT '最后登录 IP',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='管理员表';
```

**字段说明**:

| 字段            | 类型     | 长度 | 允许 NULL | 默认值            | 说明                    |
| --------------- | -------- | ---- | --------- | ----------------- | ----------------------- |
| id              | INT      | -    | NO        | AUTO_INCREMENT    | 主键                    |
| username        | VARCHAR  | 100  | NO        | -                 | 用户名（唯一）          |
| password        | VARCHAR  | 255  | NO        | -                 | BCrypt 加密密码         |
| email           | VARCHAR  | 100  | YES       | NULL              | 邮箱                    |
| role            | ENUM     | -    | NO        | 'admin'           | 角色：super_admin/admin |
| status          | TINYINT  | -    | NO        | 1                 | 1-启用/0-禁用           |
| last_login_time | DATETIME | -    | YES       | NULL              | 最后登录时间            |
| last_login_ip   | VARCHAR  | 50   | YES       | NULL              | 最后登录 IP             |
| create_time     | DATETIME | -    | NO        | CURRENT_TIMESTAMP | 创建时间                |
| update_time     | DATETIME | -    | NO        | CURRENT_TIMESTAMP | 更新时间                |

**索引**:

| 索引名      | 字段     | 类型     | 说明           |
| ----------- | -------- | -------- | -------------- |
| PRIMARY     | id       | 主键索引 | 聚簇索引       |
| uk_username | username | 唯一索引 | 用户名唯一约束 |

**默认数据**:

```sql
INSERT INTO admins (username, password, email, role, status) 
VALUES ('admin', '$2a$10$...', 'admin@example.com', 'super_admin', 1);
-- 密码：admin123 (BCrypt 加密)
```

### 2. 操作日志表 (operation_logs)

**表名**: `operation_logs`  
**说明**: 管理员操作日志表（当前实现）  
**引擎**: InnoDB  
**字符集**: utf8mb4

```sql
CREATE TABLE operation_logs (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '日志 ID',
    admin_id INT COMMENT '管理员 ID',
    action VARCHAR(50) NOT NULL COMMENT '操作类型',
    target_type VARCHAR(50) COMMENT '目标对象类型',
    target_id INT COMMENT '目标对象 ID',
    detail TEXT COMMENT '操作详情',
    ip VARCHAR(50) COMMENT '操作 IP',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (admin_id) REFERENCES admins(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='操作日志表';
```

**字段说明**:

| 字段 | 类型 | 长度 | 允许 NULL | 默认值 | 说明 |
|------|------|------|-----------|--------|------|
| id | INT | - | NO | AUTO_INCREMENT | 主键 |
| admin_id | INT | - | YES | NULL | 外键：admins.id |
| action | VARCHAR | 50 | NO | - | 操作类型 |
| target_type | VARCHAR | 50 | YES | NULL | 目标对象类型 |
| target_id | INT | - | YES | NULL | 目标对象 ID |
| detail | TEXT | - | YES | NULL | 操作详情 |
| ip | VARCHAR | 50 | YES | NULL | 操作 IP 地址 |
| create_time | DATETIME | - | NO | CURRENT_TIMESTAMP | 日志时间 |

**索引**:

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| PRIMARY | id | 主键索引 | 聚簇索引 |
| idx_operation_logs_action | action | 普通索引 | 按动作查询 |
| idx_operation_logs_target_type | target_type | 普通索引 | 按目标类型查询 |
| idx_operation_logs_create_time | create_time | 普通索引 | 时间排序 |
| idx_operation_logs_admin_time | admin_id, create_time | 复合索引 | 管理员操作时间线 |

**外键约束**:

```sql
ALTER TABLE operation_logs 
ADD CONSTRAINT fk_logs_admin 
FOREIGN KEY (admin_id) REFERENCES admins(id) ON DELETE SET NULL;
```

> 注：文末“数据库修改记录”中仍可能出现历史名称 `system_logs`，该部分保留迁移痕迹；现行表名以 `operation_logs` 为准。

### 4. 菜谱表 (recipes)

**表名**: `recipes`  
**说明**: 菜谱信息表（核心业务表）  
**引擎**: InnoDB  
**字符集**: utf8mb4

```sql
CREATE TABLE recipes (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '菜谱 ID',
    old_id INT COMMENT '原始菜谱 ID（数据迁移用）',
    title VARCHAR(200) NOT NULL COMMENT '菜谱标题',
    author VARCHAR(100) COMMENT '作者名称',
    author_uid VARCHAR(50) COMMENT '作者用户 ID',
    description TEXT COMMENT '菜谱描述',
    tips TEXT COMMENT '烹饪小贴士',
    cookware VARCHAR(200) COMMENT '所需厨具',
    image VARCHAR(255) COMMENT '封面图 URL',
    taste_id INT COMMENT '口味 ID',
    technique_id INT COMMENT '工艺 ID',
    time_cost_id INT COMMENT '耗时 ID',

    reply_count INT DEFAULT 0 COMMENT '评论数',

    rating_count INT DEFAULT 0 COMMENT '浏览数',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    FOREIGN KEY (taste_id) REFERENCES tastes(id),
    FOREIGN KEY (technique_id) REFERENCES techniques(id),
    FOREIGN KEY (time_cost_id) REFERENCES time_costs(id),
    FOREIGN KEY (difficulty_id) REFERENCES difficulties(id),
```

```sql
CREATE TABLE comments (
```

## 索引设计说明

### 索引优化原则

1. **主键索引**: 所有表都有主键，使用 InnoDB 的聚簇索引
2. **唯一索引**: 确保数据唯一性（如 username、name）
3. **外键索引**: 优化 JOIN 查询性能
4. **查询索引**: 针对高频查询条件创建索引

### 索引使用场景

#### 1. 菜谱查询优化

**场景**: 按口味、工艺、难度筛选菜谱

```sql
SELECT * FROM recipes 
WHERE taste_id = 1 
  AND difficulty_id = 2
ORDER BY create_time DESC
LIMIT 0, 10;
```

**使用的索引**:
- `idx_recipe_taste` (taste_id)
- `idx_recipe_difficulty` (difficulty_id)

#### 2. 评论查询优化

**场景**: 查询某菜谱的所有评论

```sql
SELECT c.*, u.username, u.avatar 
FROM comments c
LEFT JOIN users u ON c.user_id = u.id
WHERE c.recipe_id = 10
ORDER BY c.publish_time DESC
LIMIT 0, 10;
```

**使用的索引**:
- `idx_comment_recipe` (recipe_id)
- `idx_comment_user` (user_id) - JOIN 优化

#### 3. 用户互动查询优化

**场景**: 查询用户的收藏列表

```sql
SELECT i.recipe_id, r.title, r.image
FROM interactions i
JOIN recipes r ON i.recipe_id = r.id
WHERE i.user_id = 100 AND i.interaction_type = 'favorite'
ORDER BY i.create_time DESC;
```

**使用的索引**:
- `idx_interaction_user` (user_id)
- `idx_interaction_recipe` (recipe_id)

#### 4. 统计查询优化

**场景**: 统计各分类的菜谱数量

```sql
SELECT c.name, COUNT(rc.recipe_id) as recipe_count
FROM categories c
LEFT JOIN recipe_categories rc ON c.id = rc.category_id
GROUP BY c.id, c.name;
```

**使用的索引**:
- `uk_recipe_category` (recipe_id, category_id)
- `categories.pk` (id)

### 索引性能分析

| 索引类型 | 查询速度提升 | 空间开销 | 维护成本 |
| -------- | ------------ | -------- | -------- |
| 主键索引 | -            | 低       | 低       |
| 唯一索引 | 高           | 低       | 中       |
| 外键索引 | 高           | 中       | 中       |
| 普通索引 | 中           | 中       | 低       |

### 索引创建建议

1. **避免过度索引**: 每个索引都会影响写入性能
2. **定期分析索引**: 使用 `EXPLAIN` 分析查询计划
3. **删除无用索引**: 长期未使用的索引应及时删除
4. **组合索引**: 对于多条件查询，考虑创建组合索引

### 2026-03-03 - 修复数据库字段错误

**修改人**: AI Assistant

   - 解决: 将 `r.view_count` 改为 `r.rating_count as view_count`

   - 问题: BCrypt 加密密码约 60 字符，部分数据库 password 列长度可能不足
   - 解决: 执行 `ALTER TABLE users MODIFY COLUMN password VARCHAR(255);`
   - 说明: 确保 password 列能存储 BCrypt 加密后的完整密码

3. **修复 UserServiceImpl 密码加密方式不一致**
   - 问题: 前端用户登录使用 MD5，管理后台创建用户使用 BCrypt，导致密码不兼容
   - 解决: 将 UserServiceImpl 中的 MD5 加密改为 BCrypt
   - 说明: 统一使用 BCrypt 加密方式

4. **添加 recipes 表 status 列**
   - 问题: RecipeMapper 查询使用了 `r.status`，但 recipes 表没有该列
   - 解决: 执行 `ALTER TABLE recipes ADD COLUMN status TINYINT DEFAULT 1;`
   - 说明: 支持菜谱状态管理（1-正常，0-禁用）
