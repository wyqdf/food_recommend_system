# 快速启动指南

> 目标：让第一次从 GitHub 拉取 `foodrec` 的人，在本地最短路径把项目跑起来。  
> 当前核对日期：`2026-03-19`  
> 推荐启动口径：`前端 3000 + 后端 8081 + MySQL 本地`  
> 可选完整体验：再加一个 `Elasticsearch 9200`

## 一、先明确 3 个事实

1. 这个项目的用户端和管理后台都在同一个 `frontend/` 工程里，不存在独立 `admin/` 前端。
2. 后端默认不会自动执行 `schema.sql` / `data.sql`，数据库需要你手动初始化。
3. 第一次跑项目，不需要先上 Elasticsearch；默认 `SEARCH_ENGINE=mysql` 就能先把前后端跑起来。

请先确认本机有这些环境：

- Java 21

## 三、最短路径：先把项目跑起来

### 步骤 1：克隆仓库

```bash
git clone git@github.com:wyqdf/food_recommend_system.git
cd food_recommend_system
```

如果你是直接下载 ZIP 或本地已有目录，只要确保当前目录里有：

- `backend/let-me-cook`
- `frontend`
- `readme.md`
- `快速启动指南.md`

### 步骤 2：初始化 MySQL

项目默认假设数据库是：

- 数据库名：`food_recommend`
- 用户名：`root`
- 密码：`123456`

先创建数据库：

```bash
mysql -uroot -p -e "CREATE DATABASE IF NOT EXISTS food_recommend DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

再导入结构和种子数据：

```bash
mysql -uroot -p food_recommend < backend/let-me-cook/src/main/resources/schema.sql
mysql -uroot -p food_recommend < backend/let-me-cook/src/main/resources/data.sql
```

说明：

- `schema.sql`：建表、索引、触发器
- `data.sql`：基础数据、默认账号、演示数据
- `data.sql` 适合首次导入；如果要重复导入，建议先清空数据库

### 步骤 3：启动后端

```bash
cd backend/let-me-cook
mvn spring-boot:run
```

启动成功后，后端默认监听：

- `http://127.0.0.1:8081`

快速自检：

```bash
curl http://127.0.0.1:8081/api/categories
```

如果能返回 JSON，说明后端已经跑起来了。

### 步骤 4：启动前端

另开一个终端：

```bash
cd frontend
npm install
npm run dev
```

前端默认监听：

- `http://127.0.0.1:3000`

### 步骤 5：打开页面

启动完成后，直接访问：

- 用户端首页：`http://127.0.0.1:3000`
- 推荐页：`http://127.0.0.1:3000/recommend`
- 搜索页：`http://127.0.0.1:3000/search`
- 管理员登录：`http://127.0.0.1:3000/admin/login`

## 六、如果你的数据库密码不是 123456

后端默认读取：

- 用户名：`root`
- 密码：`${DB_PASSWORD:123456}`

所以如果你的 MySQL `root` 密码不是 `123456`，启动后端前设置环境变量即可。

### PowerShell

```powershell
$env:DB_PASSWORD='你的MySQL密码'
cd backend/let-me-cook
mvn spring-boot:run
```

### macOS / Linux

```bash
export DB_PASSWORD='你的MySQL密码'
cd backend/let-me-cook
mvn spring-boot:run
```

如果你连用户名、端口、数据库名都不同，可以进一步覆盖：

- `SPRING_DATASOURCE_URL`
- `SPRING_DATASOURCE_USERNAME`

## 七、可选：启用 Elasticsearch（完整体验 Search V2）

如果你只是想把项目跑起来，可以先跳过这一节。  
如果你想体验当前完整搜索能力，再继续。

### 步骤 1：启动 ES

在仓库根目录执行：

```bash
docker compose up -d elasticsearch
```

启动后访问：

- `http://127.0.0.1:9200`

健康检查：

```bash
curl http://127.0.0.1:9200/_cluster/health
```

### 步骤 2：用 ES 模式启动后端

默认后端是 `SEARCH_ENGINE=mysql`。  
如果要切到 ES，请显式设置环境变量。

#### PowerShell

```powershell
$env:SEARCH_ENGINE='elasticsearch'
cd backend/let-me-cook
mvn spring-boot:run
```

#### macOS / Linux

```bash
export SEARCH_ENGINE=elasticsearch
cd backend/let-me-cook
mvn spring-boot:run
```

### 步骤 3：首次重建搜索索引

第一次切到 ES 后，索引里还没有数据，需要做一次全量重建。

最简单的方式：

1. 登录管理后台：`http://127.0.0.1:3000/admin/login`
2. 进入数据看板
3. 点击“搜索索引重建”

完成后，搜索页就会走 Search V2。

## 八、推荐的本地开发方式

### 方式 A：推荐，最稳

- MySQL 本地
- 后端本地：`8081`
- 前端本地：`3000`
- ES 按需启动

适合：

- 第一次上手
- 本地调试
- 改代码时热更新

### 方式 B：和当前团队演示环境更接近

- MySQL 本地
- 后端宿主机：`8081`
- 前端 Docker：`5173`
- ES Docker：`9200`

适合：

- 做演示
- 验证 Docker 前端 Nginx 代理

但第一次接手不建议直接从方式 B 开始，因为更容易遇到“旧容器 / 旧静态资源 / 端口被占用”的问题。

## 十、最常见的 6 个问题

### 1. 页面打开了，但接口全 500

大概率是 MySQL 没初始化，或者密码不对。

优先检查：

- `food_recommend` 是否存在
- `schema.sql` / `data.sql` 是否导入
- `DB_PASSWORD` 是否正确

### 2. 后端启动成功，但管理员无法登录

大概率是你没有导入 `data.sql`，或者导入后又清空了 `admins` 表。

默认管理员来自：

- `backend/let-me-cook/src/main/resources/data.sql`

### 3. 前端能打开，但没有数据

先看后端接口：

```bash
curl http://127.0.0.1:8081/api/categories
```

如果这里就不通，问题不在前端。

### 4. 搜索不好用或报错

第一次本地启动建议先接受默认 `SEARCH_ENGINE=mysql`。

如果你切到了 ES 模式，还需要：

1. 确认 `9200` 正常
2. 做一次搜索索引重建

### 5. 为什么我看到的是 3000，不是 5173

因为 `npm run dev` 的 Vite 默认端口就是 `3000`。  
`5173` 是团队当前经常使用的 Docker 前端演示端口，不是第一次本地开发的推荐入口。

### 6. 为什么我导入了 schema.sql，还是缺字段或索引

因为数据库真实结构不仅来自 `schema.sql`，启动后端时 `DatabaseMigrationConfig` 还会补表、补列、补索引、隐藏一部分冗余索引。
