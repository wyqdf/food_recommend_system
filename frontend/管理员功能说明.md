# 管理员功能使用说明

## 一、功能概述

管理员后台系统已创建以下功能模块：

### 1. 用户管理
- 用户列表展示（支持分页）
- 用户搜索（按用户名/昵称）
- 用户状态筛选（正常/禁用）
- 新增用户
- 编辑用户
- 删除用户（支持批量删除）
- 禁用/启用用户

### 2. 食谱管理
- 食谱列表展示（支持分页）
- 食谱搜索（按标题）
- 食谱难度筛选
- 新增食谱
- 编辑食谱
- 删除食谱（支持批量删除）
- 查看食谱详情

### 3. 分类管理
- 分类列表展示
- 新增分类
- 编辑分类
- 删除分类
- 显示每个分类的食谱数量

### 4. 数据统计
- 数据概览卡片（总用户数、总食谱数等）
- 今日数据统计
- 新增用户趋势图
- 食谱分类分布饼图
- 食谱难度分布柱状图
- 热门食谱排行榜

### 5. 系统日志
- 操作日志列表
- 日志筛选（管理员、模块）
- 分页展示

## 二、访问方式

### 1. 启动项目

```bash
cd frontend
npm install
npm run dev
```

- 管理后台登录页：`http://localhost:5173/admin/login`
- 管理后台首页：`http://localhost:5173/admin/dashboard`

## 四、API 接口说明

所有 API 接口都在 `frontend/src/api/admin.js` 中定义，目前使用 mock 数据：

- `adminApi`: 管理员认证相关
- `adminUserApi`: 用户管理相关
- `adminRecipeApi`: 食谱管理相关
- `adminCategoryApi`: 分类管理相关
- `adminStatisticsApi`: 统计数据相关
- `adminLogApi`: 系统日志相关

## 六、后端实现状态

### 1. 数据库表 ✅
- ✅ admins 表已创建（含默认管理员账号：admin/admin123）
- ✅ system_logs 表已创建
- ✅ users 表已增加字段（status, phone, last_login_time）
- ✅ 相关索引已创建

### 2. 待实现的 API 接口
需要实现 5 个 Controller 类，共 26 个接口：
- **AdminAuthController**: 3 个接口（登录、获取信息、修改密码）
- **UserManagementController**: 7 个接口（用户 CRUD、批量操作、状态切换）
- **RecipeManagementController**: 7 个接口（食谱 CRUD、批量操作、审核）
- **CategoryManagementController**: 4 个接口（分类 CRUD）
- **StatisticsController**: 5 个接口（数据统计、日志列表）

详细接口列表请参考 `backend/管理员 API 设计文档.md`

### 3. 开发优先级
1. **第一阶段**：AdminAuthController + AdminAuthInterceptor（认证基础）
2. **第二阶段**：UserManagementController（用户管理）
3. **第三阶段**：RecipeManagementController + CategoryManagementController（可并行）
4. **第四阶段**：StatisticsController（统计数据）

## 九、注意事项

1. 所有数据目前都是 mock 数据，刷新后会重置
2. 删除操作有二次确认提示
3. 表单提交前会进行验证
4. 路由守卫会检查管理员登录状态
5. 需要安装 ECharts: `npm install echarts@^5.4.3`
