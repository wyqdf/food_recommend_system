# 管理员功能开发日志

## 版本历史

### v2.0 - 2026-03-01

#### 新增功能

1. **食谱管理全面增强**
   - ✅ 支持编辑所有食谱属性
     - 基本信息：标题、作者、作者 UID、封面图 URL
     - 属性选择：难度、口味、技法、耗时（从数据库加载的下拉选项）
     - 其他信息：厨具、描述、小贴士
     - 关联数据：食材清单、烹饪步骤、分类
   
   - ✅ 食材管理
     - 动态添加/删除食材
     - 每个食材可设置类型（主料/辅料/调料）
     - 可设置用量
   
   - ✅ 步骤管理
     - 动态添加/删除烹饪步骤
     - 每个步骤可配图
   
   - ✅ 分类管理
     - Checkbox 多选分类

2. **用户密码管理**
   - ✅ 密码重置功能
     - 管理员可以为用户重置密码
     - 密码长度验证（至少 6 位）
     - 两次密码确认验证
     - 独立的密码修改对话框

3. **属性管理（新增页面）**
   - ✅ 口味管理 - 管理咸鲜、酸甜、麻辣等口味
   - ✅ 烹饪技法管理 - 管理炒、蒸、煮、炸、炖等技法
   - ✅ 耗时管理 - 管理 15 分钟、30 分钟等耗时选项
   - ✅ 难度管理 - 管理简单、中等、困难等难度
   - ✅ 食材管理 - 管理猪肉、牛肉、鸡肉等食材

4. **删除保护机制**
   - ✅ 智能删除检查
     - 显示每个属性关联的食谱数量
     - 已被使用的属性无法删除
     - 删除时显示警告提示
     - 只有 `recipeCount = 0` 的属性才能删除

#### 数据库字段映射

**食谱表 (recipes)**:
```sql
- id: 主键
- title: 标题
- author: 作者
- author_uid: 作者 UID
- image: 封面图
- description: 描述
- tips: 小贴士
- cookware: 厨具
- taste_id: 口味 ID (FK -> tastes.id)
- technique_id: 技法 ID (FK -> techniques.id)
- time_cost_id: 耗时 ID (FK -> time_costs.id)
- difficulty_id: 难度 ID (FK -> difficulties.id)
```

**关联表**:
- `recipe_ingredients`: 食谱食材关联
- `recipe_categories`: 食谱分类关联
- `recipe_steps`: 食谱步骤

**属性表**:
- `tastes`: 口味表
- `techniques`: 技法表
- `time_costs`: 耗时表
- `difficulties`: 难度表
- `ingredients`: 食材表

## 开发注意事项

### 1. Mock 数据与真实 API 的切换

当前使用 Mock 数据，后续对接真实 API 时：

```javascript
// 当前（Mock）
import { mockDelay } from '@/utils/mock';

// 对接真实 API
import request from '@/utils/request';

export function getUserList(params) {
  return request({
    url: '/admin/users',
    method: 'get',
    params
  });
}
```

### 2. 表单验证

所有表单都配置了验证规则：

```javascript
const rules = {
  title: [
    { required: true, message: '请输入标题', trigger: 'blur' }
  ],
  // ... 其他字段
};
```

### 3. 错误处理

统一的错误处理：

```javascript
try {
  await apiFunction();
  ElMessage.success('操作成功');
} catch (error) {
  ElMessage.error(error.message || '操作失败');
}
```

### 4. 权限控制

路由守卫保护：

```javascript
{
  path: '/admin',
  component: AdminLayout,
  children: [
    {
      path: 'dashboard',
      component: () => import('@/views/admin/Dashboard.vue'),
      meta: { requiresAdminAuth: true }
    }
  ]
}
```
